Vix 语言官方工具

Very

Vix语言包管理器兼项目管理工具

GitHub 仓库 安装 Very 
# Very 命令帮助
very --help

very add   fexcode.vnet
very init  my-project
very search vnet
very list  --tree

very tool add   game
very tool search game

概述

Very 是 Vix 编程语言的官方项目管理与构建工具。它依托 Git 仓库进行包管理，提供简洁高效的命令语法，支持包依赖管理、工具链管理和项目初始化。

Git 原生集成

直接基于 Git 仓库管理依赖，无需中心化包注册服务器。支持 GitHub、Gitee 等主流 Git 平台，灵活指定分支和版本。

简洁的简写语法

内置智能包名展开规则：fexcode.vnet 自动映射为 github.com/fexcode/vnet，@ 前缀默认指向 Gitee，省去繁琐的完整 URL 输入。

包与工具双管理

统一管理项目依赖（vlib-*）和可执行工具（vtool-*）。工具自动编译为原生可执行文件，支持增量更新和自动安装。

包名缩写规则

Very 提供了一套极具创意且高效的包名缩写系统。无需输入完整 Git URL，只需记住几条简单规则，即可用最少的字符定位任意仓库。

基本规则
user.repo → github.com/user/repo

以 . 分隔用户名和仓库名，默认托管在 GitHub。这是最常用的写法。

指定分支
user.repo@branch → github.com/user/repo @ branch

在包名后追加 @分支名 即可锁定特定分支，默认跟踪 main 或 master。

指定平台
gitee.com:user.repo → gitee.com/user/repo

用 域名: 前缀显式指定 Git 托管平台，Very 自动将 : 转换为 /。

平台简写
gitee:user.repo → gitee.com/user/repo

.com 后缀可省略。Very 自动补全常见顶级域名，输入更快捷。

Gitee 快捷符
@user.repo → gitee.com/user/repo

@ 前缀是 Gitee 的快捷入口——一个字符切换平台，对国内开发者尤为友好。

裸名称 — 官方包
vnet → github.com/vixlang/vlib-vnet

不含 . 的单个词为"裸名称"。Very 自动匹配官方仓库，库前缀 vlib-，工具前缀 vtool-。

创意组合： 以上规则可自由组合。例如 @user.repo@dev 表示 Gitee 上 user/repo 的 dev 分支； gitee:user.repo@master 是完整显式写法。Very 的解析器按 平台 → 用户 → 仓库 → 分支 的顺序智能解构，让你用最少的按键表达最精确的意图。

安装 Very

Very 使用 Python 开发，通过 uv 工具即可快速安装。

1

确保已安装 Python 和 uv 包管理器

# 安装 uv（如未安装）
pip install uv
2

克隆 Very 仓库并安装

git clone https://github.com/vixlang/Very.git
cd Very
uv tool install .
3

验证安装

very --version
very --help

项目配置

每个 Vix 项目通过 vindex.toml 文件描述元数据、依赖关系和构建配置。

vindex.toml

项目配置文件，定义项目名称、版本、作者、依赖和构建选项。Very 通过此文件识别和管理项目。

查看配置文档

项目结构

使用 very init 创建的标准项目骨架，包含入口文件、配置文件和 gitignore。

my-project/
├── vindex.toml # 项目配置
├── main.vix # 入口文件
├── .gitignore
└── README.md
